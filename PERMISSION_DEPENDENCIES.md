# WMS 系统权限依赖关系表

本文档整理了 WMS 系统中各个业务模块操作所需的权限依赖关系。
在分配角色权限时，请参考此表，确保用户拥有执行特定操作所需的所有基础数据访问权限。
系统中已实现针对以下核心业务的自动勾选联动。

## 核心业务模块依赖

| 业务模块 | 权限字符 (Key) | 依赖的权限 (Dependencies) | 说明 |
| :--- | :--- | :--- | :--- |
| **入库管理** | `wms:inbound` | 1. `base:warehouse:list` (仓库列表)<br>2. `base:product:list` (产品列表)<br>3. `base:supplier:list` (供应商列表) | 创建入库单时，必须能够选择仓库、产品和供应商。 |
| **出库管理** | `wms:outbound` | 1. `base:warehouse:list` (仓库列表)<br>2. `base:product:list` (产品列表)<br>3. `base:customer:list` (客户列表)<br>4. `wms:inventory:list` (库存查询) | 创建出库单时，需选择客户、产品和仓库，并校验库存数量。 |
| **库存查询** | `wms:inventory` | 1. `base:warehouse:list` (仓库列表)<br>2. `base:product:list` (产品列表) | 查询库存时，通常需要关联显示仓库名称和产品详情。 |

## 基础数据模块说明

基础数据模块通常作为其他业务模块的前置依赖，但也可能存在内部依赖。

| 基础模块 | 权限字符 | 典型用途 |
| :--- | :--- | :--- |
| **仓库管理** | `base:warehouse` | 入库、出库、库存查询的核心依赖。 |
| **产品管理** | `base:product` | 所有涉及物料流转（入库、出库、库存）的核心依赖。 |
| **客户管理** | `base:customer` | **出库**业务的必要依赖。 |
| **供应商管理** | `base:supplier` | **入库**业务的必要依赖。 |
| **库位/库区/货架** | `base:location`等 | 通常依赖于仓库权限 (`base:warehouse`) 才能进行层级筛选。 |

## 系统管理权限

| 模块 | 权限字符 | 说明 |
| :--- | :--- | :--- |
| **用户管理** | `system:user` | 依赖 `system:role:list` (角色列表) 以便在创建用户时分配角色。 |
| **角色管理** | `system:role` | 依赖 `system:menu:list` (菜单列表) 以便在创建角色时分配权限。 |

---
*注：前端角色管理页面已根据此表实现了部分自动联动功能，当勾选左侧业务权限时，右侧依赖将自动被选中。*
